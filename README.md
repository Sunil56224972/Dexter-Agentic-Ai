## <img src="https://media.giphy.com/media/l41lVsYDBC0UVQJCE/giphy.gif" width="30"> About Dexter Agentic AI

Dexter Agentic AI is an **autonomous terminal AI agent** designed to perform real system operations.

Unlike traditional chatbots, Dexter can:

• execute terminal commands
• explore filesystem directories
• create and edit files
• interact with users through an AI reasoning engine

This transforms AI from a **passive assistant into an active operator.**

## <img src="https://media.giphy.com/media/3o7aCTfyhYawdOXcFW/giphy.gif" width="30"> Architecture

Dexter Agentic AI contains three main layers.

### 🧠 AI Brain

Uses **Google Gemini AI** for reasoning and decision making.

Example tools

```bash
execute_terminal_command
list_directory
write_file
```

### ⚙️ Tool Execution

Example:

User request

```bash
show files
```

AI internally selects

```bash
list_directory
```

Then executes and returns results.

### 🔐 Safety Layer

Before running commands Dexter asks permission.

```bash
⚠️ Run command? (y/n)
```

## ⚔️ Dexter Agentic AI vs Claude Code

Dexter Agentic AI is designed as a **lightweight terminal AI operator**, while Claude Code is a **large-scale coding assistant built by Anthropic**.

| Feature                | Dexter Agentic AI            | Claude Code             |
| ---------------------- | ---------------------------- | ----------------------- |
| Core idea              | Autonomous terminal AI agent | AI coding assistant     |
| Runs terminal commands | ✅ Yes                        | ⚠️ Limited              |
| File system control    | ✅ Full directory control     | ⚠️ Mostly code editing  |
| Tool based execution   | ✅ Yes                        | ❌ Mostly conversational |
| Lightweight setup      | ✅ Very lightweight           | ❌ Heavy infrastructure  |
| Customizable           | ✅ Fully customizable         | ⚠️ Limited              |

### Philosophy Difference

**Dexter Agentic AI**

```text
Observe → Decide → Execute
```

Dexter acts like an **AI operator controlling tools inside the terminal.**

**Claude Code**

```text
Understand → Suggest → Generate
```

Claude Code focuses more on **AI assisted coding and code generation**.

## <img src="https://media.giphy.com/media/5VKbvrjxpVJCM/giphy.gif" width="30"> Installation

⚙️ Prerequisites

Before running Dexter AI, make sure Node.js is installed.

Install Node.js

Go to the official website
https://nodejs.org

Download the LTS version (Recommended).

Install it like normal software.

Verify installation by running:

node -v

and

npm -v

If both commands show versions, Node.js is installed correctly.

Clone repository

```bash
git clone https://github.com/Sunil56224972/dexter-agentic-ai.git
cd dexter-agentic-ai
```

Install dependencies

```bash
npm install
```

Create `.env`

```env
GEMINI_API_KEY=your_api_key_here
```

Run Dexter

```bash
dexter
```

## 🔐 Dexter Agentic AI & Computer Privacy

<p align="center">
<img src="https://readme-typing-svg.herokuapp.com?font=Orbitron&size=26&duration=2500&pause=1000&color=00FF9C&center=true&vCenter=true&width=800&lines=Dexter+Protects+Your+System;Privacy+Focused+AI+Agent;Secure+Command+Execution;Directory+Access+With+User+Permission"/>
</p>

Dexter Agentic AI is designed to **interact with the computer safely while respecting user privacy**.

Because Dexter can access **files, directories, and terminal commands**, privacy and security are extremely important.

Dexter follows a **permission-based execution model**, ensuring that sensitive actions are never performed automatically.

---

### 🧠 Privacy-First Design

<p align="center">
<img src="https://readme-typing-svg.herokuapp.com?font=Fira+Code&size=22&duration=2500&pause=1000&color=00ff9c&center=true&vCenter=true&width=700&lines=Permission+Before+Execution;User+Control+Over+Commands;Safe+Directory+Access;Transparent+AI+Actions"/>
</p>

Dexter follows these privacy principles:

• commands require **explicit user approval**
• directory access is **visible and controlled**
• no hidden background actions
• full transparency in terminal output

This ensures users always know **what the AI is doing on their system**.

---

### 📁 Directory Privacy

Dexter interacts with directories using controlled tools such as:

```bash
list_directory
write_file
execute_terminal_command
```

Directory operations follow strict rules:

| Rule                 | Explanation                             |
| -------------------- | --------------------------------------- |
| User Permission      | Commands require approval               |
| Transparent Actions  | All operations are visible in terminal  |
| Controlled Access    | Only requested directories are accessed |
| Safe File Operations | Files are modified only when instructed |

---

### ⚠️ Command Safety System

<p align="center">
<img src="https://readme-typing-svg.herokuapp.com?font=Fira+Code&size=22&duration=2500&pause=1200&color=ff3c3c&center=true&vCenter=true&width=700&lines=Command+Verification+Enabled;Permission+Prompt+Activated;Unauthorized+Execution+Blocked"/>
</p>

Before executing terminal commands, Dexter asks the user:

```bash
⚠️ Run: 'command'? (y/n)
```

This mechanism prevents:

• accidental system changes
• unauthorized command execution
• dangerous operations without confirmation

---

### 🛡️ Secure Agent Behavior

Dexter operates using a **human-in-the-loop security model**.

This means the user always stays in control.

Security features include:

• permission-based command execution
• transparent terminal logs
• safe directory listing
• controlled file writing
